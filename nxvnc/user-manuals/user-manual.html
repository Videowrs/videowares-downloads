videowares WebVNC

User Manual Version 1.0.0 Last updated: March 2026

1. Introduction

videowares WebVNC is a web-based remote desktop client that lets you access VNC servers directly from a Chromium-based browser (Chrome, Edge, Brave, etc.) with no browser plugins or extensions required.

Key Features

  • Browser-based — No client software to install on the viewing machine
  • Secure — HTTPS/TLS encryption, JWT authentication, CSRF protection
  • Multi-user — Role-based access control (Admin, Operator, Viewer)
  • Windows Service — Runs at boot without user login
  • System Tray — Control panel for quick service management
  • Audit Logging — Track all user activity
  • Saved Connections — Store and share VNC connection profiles

How It Works

Browser (WebVNC Client) | | HTTPS / WebSocket (WSS) v WebVNC Server (port 6080) | | TCP v VNC Server (e.g., TightVNC on port 5900)

The WebVNC server acts as a secure bridge between your browser and VNC servers on the network. The browser connects to WebVNC over encrypted HTTPS/WebSocket, and WebVNC proxies the connection to the target VNC server over TCP.

3. Installing TightVNC Server

TightVNC is a free, lightweight VNC server for Windows. Install it on every machine you want to control remotely through WebVNC.

3.1 Download TightVNC

  1. Visit https://www.tightvnc.com/download.php
  2. Download the TightVNC Installer for Windows (64-bit recommended)
  3. The file will be named something like tightvnc-2.8.85-gpl-setup-64bit.msi

3.2 Run the Installer

  1. Double-click the downloaded .msi file
  2. Click Next on the Welcome screen
  3. Accept the license agreement and click Next
  4. On the Choose Setup Type screen, select:
    • Custom if you only need the VNC Server (recommended for remote machines)
    • Typical if you want both Server and Viewer components

3.3 Custom Installation (Server Only)

If you selected Custom:

  1. Click on TightVNC Viewer and select Entire feature will be unavailable (WebVNC replaces the viewer)
  2. Keep TightVNC Server enabled
  3. Click Next

3.4 Configure TightVNC Server

During installation, you will be prompted to configure the server:

Set Passwords

  • Password for Remote Access: This is the VNC connection password. Set a strong password and remember it — you will need it when connecting from WebVNC.
  • Administrative Password: Used to change TightVNC settings later. Can be the same or different.

Important: The remote access password is what you will enter in WebVNC when creating a connection profile.

Recommended Settings

SettingRecommended ValueNotes
Accept connections on port5900Default VNC port
Require VNC authenticationCheckedAlways require a password
Allow loopback connectionsCheckedAllows local testing
Enable file transfersUncheckedNot supported by WebVNC
Run as serviceCheckedStart VNC at Windows boot
Block remote inputUncheckedUnless you want view-only

3.5 Complete Installation

  1. Click Next and then Install
  2. Wait for installation to complete
  3. Click Finish

TightVNC Server will start automatically as a Windows Service.

3.6 Verify TightVNC is Running

  1. Look for the TightVNC icon in the Windows system tray (bottom-right, near the clock).
  2. Alternatively, open Services (press Win+R, type services.msc, press Enter) and find TightVNC Server. Status should be Running.

3.7 Configure Windows Firewall for TightVNC

TightVNC typically configures the firewall during installation. If remote connections fail, open an elevated PowerShell and run:

New-NetFirewallRule -DisplayName "TightVNC Server" `
  -Direction Inbound -Protocol TCP -LocalPort 5900 `
  -Action Allow -Profile Domain,Private

3.8 TightVNC on Multiple Machines

Repeat the installation on every Windows machine you want to access remotely. Each machine will run TightVNC Server on port 5900 and be accessible from WebVNC by its IP address or hostname.

Tip: For non-Windows machines, use TigerVNC (Linux) or the built-in Screen Sharing (macOS). The VNC protocol is cross-platform.

4. Installing videowares WebVNC

4.1 Using the Installer (Recommended)

  1. Obtain the installer file: WebVNCSetup_x.x.x.xxxxx.exe
  2. Right-click and select Run as administrator
  3. Follow the installation wizard

Installation Options

OptionDescriptionDefault
Start the service nowStarts WebVNC immediately after installChecked
Start tray on loginAdds tray icon to Windows startupChecked
Create desktop iconShortcut on desktopUnchecked

The installer will:

  • Copy files to C:\Program Files\videowares WebVNC\
  • Create data directories at C:\ProgramData\WebVNC\
  • Register and configure the Windows Service
  • Set up log rotation
  • Optionally start the service and tray app

Installation Directories

PathContents
C:\Program Files\videowares WebVNC\Application executables, client files, configuration
C:\ProgramData\WebVNC\data\SQLite database (webvnc.db)
C:\ProgramData\WebVNC\data\tls\Auto-generated TLS certificates
C:\ProgramData\WebVNC\logs\Service log files
%LocalAppData%\WebVNC\Tray application logs

4.2 Manual Installation (Without Installer)

  1. Copy the dist\WebVNC\ folder to your desired location (e.g., C:\WebVNC\)
  2. Open an Administrator Command Prompt
  3. Navigate to the folder and run: 
    manage_service.bat install
manage_service.bat start
  4. Optionally, run WebVNC.exe for the system tray control panel

4.3 Uninstalling

  1. Open Settings > Apps > Installed Apps
  2. Find videowares WebVNC and click Uninstall

Note: The database at C:\ProgramData\WebVNC\data\ is not removed during uninstall to preserve your configuration. Delete it manually for a clean removal.

5. First-Time Setup

5.1 Access the Web Interface

  1. Open a Chromium-based browser (Chrome, Edge, Brave)
  2. Navigate to: https://localhost:6080
  3. You will see a browser security warning (self-signed certificate):
    • Chrome/Edge: Click Advanced then Proceed to localhost (unsafe)
    • This is safe — the connection is still encrypted

5.2 Log In with Default Credentials

FieldValue
Usernameadmin
Passwordadmin

Important: You will be prompted to change your password on first login. Choose a strong password (minimum 8 characters).

5.3 Create Additional Users

  1. Click Admin Panel in the sidebar
  2. Go to the Users tab
  3. Click Add User and fill in username, password, and role

User Roles

RolePermissions
AdminFull access. Manage users, connections, licenses, settings. View audit logs.
OperatorUse assigned connections. Create and manage own connections.

5.4 Add Your First VNC Connection

  1. In the sidebar, fill in the New Connection form:

    • Host: IP address of the machine running TightVNC
    • Port: 5900 (default)

  2. Click Connect

15. Security Features

15.1 Encryption

  • All traffic uses HTTPS/TLS and WSS
  • Stored VNC passwords: AES-256-GCM
  • User passwords: bcrypt hashing

15.2 Authentication

  • JWT session tokens (8-hour default)
  • HTTP-only, Secure cookies
  • CSRF protection on state-changing endpoints
  • Forced password change on first login
  • Password history enforcement

15.3 Rate Limiting

Endpoint TypeLimitWindow
Login10 attempts15 minutes
API100 requests1 minute
Admin endpoints30 requests1 minute

15.4 Access Control

  • Role-based (Admin, Operator, Viewer)
  • Connection-level assignment
  • Viewer role enforces view-only server-side

15.5 Network Security

  • SSRF protection (configurable)
  • DNS and port validation
  • HSTS headers (1-year max-age)

16. TLS Certificates

16.1 Self-Signed Certificates (Default)

  • Algorithm: RSA 2048-bit, SHA-256
  • Validity: 90 days
  • Auto-renewal: Regenerated when ≤30 days remain
  • Location: C:\ProgramData\WebVNC\data\tls\

16.2 Trusting the Self-Signed Certificate

Option A: Browser exception — Click Advanced > Proceed when prompted.

Option B: Install in Windows

  1. Navigate to https://localhost:6080 in Chrome/Edge
  2. Click padlock > Certificate > Details > Copy to File
  3. Export as .cer, double-click it > Install Certificate
  4. Select Local Machine > Trusted Root Certification Authorities
  5. Restart your browser

16.3 Certificate Expiry Monitoring

17. Firewall Configuration

17.1 WebVNC Server Rule

New-NetFirewallRule -DisplayName "videowares WebVNC" `
  -Direction Inbound -Protocol TCP -LocalPort 6080 `
  -Action Allow -Profile Domain,Private `
  -Description "Allow WebVNC web-based remote desktop access"

17.2 TightVNC Server Rule

New-NetFirewallRule -DisplayName "TightVNC Server" `
  -Direction Inbound -Protocol TCP -LocalPort 5900 `
  -Action Allow -Profile Domain,Private `
  -Description "Allow VNC connections from WebVNC server"

17.3 Restrict to Specific Subnet

New-NetFirewallRule -DisplayName "videowares WebVNC (Restricted)" `
  -Direction Inbound -Protocol TCP -LocalPort 6080 `
  -Action Allow -Profile Domain,Private `
  -RemoteAddress 192.168.1.0/24 `
  -Description "Allow WebVNC from local subnet only"

17.4 Remove a Firewall Rule

Remove-NetFirewallRule -DisplayName "videowares WebVNC"

18. Troubleshooting

18.1 Cannot Access https://localhost:6080

CheckSolution
Service running?Check tray icon or run sc.exe query WebVNC
Correct port?Check config.json for the port setting
Firewall blocking?Add firewall rule (see Section 17)
Using HTTP?WebVNC requires https:// not http://
Non-Chromium browser?Use Chrome, Edge, or Brave

18.2 Browser Security Warning

Expected with self-signed certificates. Click Advanced > Proceed. See Section 16.2 to suppress permanently.

18.3 Cannot Connect to VNC Server

CheckSolution
TightVNC running?Check TightVNC tray icon or Services
Correct IP/hostname?Ping the target from the WebVNC server
Correct port?Default is 5900; check TightVNC config
Firewall on target?Ensure port 5900 is open
Correct password?Re-enter the TightVNC password
Network connectivity?Ensure WebVNC server can reach target on TCP 5900

18.4 Session Drops After 5 Minutes

You are in free mode. Wait 5 minutes for cooldown, or upload a license in Admin Panel > License.

18.5 Service Fails to Start

Check the service log at C:\ProgramData\WebVNC\logs\service.log

ErrorSolution
Port in useChange port in config.json or stop conflicting app
Database lockedStop all WebVNC processes and restart
Permission deniedEnsure service runs as Local System

18.6 Forgot Admin Password

  1. Stop the service
  2. Delete the database: C:\ProgramData\WebVNC\data\webvnc.db
  3. Start the service (fresh database created)
  4. Log in with admin / admin

Warning: This deletes all users, connections, audit logs, and settings.

18.7 Tray Icon Not Visible

  1. Check if WebVNC.exe is running in Task Manager
  2. Click the ^ arrow in the system tray for hidden icons
  3. Right-click taskbar > Taskbar settings > Other system tray icons > Enable WebVNC

18.8 Poor Display Performance

  1. Lower Quality Level to 3-5
  2. Increase Compression Level to 5-7
  3. Switch to 8-bit color depth
  4. Enable Frame Rate Limit (30 or 15 FPS)
  5. Ensure fast network between WebVNC and VNC server

18.9 Keyboard Shortcuts Not Working

  1. Enable Keyboard Grab in Settings
  2. Use the Keys toolbar button for special combinations
  3. Ensure you are not in View Only mode

18.10 PowerShell sc Returns Wrong Output

In PowerShell, sc is an alias for Set-Content. Always use sc.exe:

# Wrong:
sc query WebVNC

# Correct:
sc.exe query WebVNC
